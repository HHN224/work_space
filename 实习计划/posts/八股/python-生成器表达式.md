## 一句话定义

生成器表达式是python中一种简洁地生成迭代器的方式，它允许你惰性地获取数据，对内存友好。

## 我的理解

生成器表达式是和列表推导式相对的概念
| 特性 | 列表推导式 `[x for x in range(5)]` | 生成器表达式 `(x for x in range(5))` |
|------|-----------------------------------|-------------------------------------|
| 返回类型 | 列表（一次性生成所有元素） | 生成器对象（惰性求值） |
| 内存占用 | 存储所有元素，占用大量内存 | 不存储元素，占用极少内存 |
| 迭代次数 | 可重复迭代 | 只能迭代一次 |
| 访问方式 | 支持索引 `lst[0]` | 不支持索引 |

## 一个例子
```python
# 列表推导式:一次性生成,占内存
lst = [x * x for x in range(5)]
print(lst)          # [0, 1, 4, 9, 16]   ← 已经是完整列表
print(lst[0])       # 0   ← 支持索引
print(lst[0])       # 0   ← 可重复访问

# 生成器表达式:惰性,不占内存
gen = (x * x for x in range(5))
print(gen)          # <generator object ...>  ← 还没算,只是个生成器
print(next(gen))    # 0   ← 现才算第一个
print(next(gen))    # 1   ← 接着算第二个

# for 也能迭代,但只能迭代一次
for x in gen:       # 4, 9, 16  ← 接着上面的,前面两个已经消费掉了
    print(x)
for x in gen:       # (无输出)  ← 第二次 for 拿不到东西了
    print(x)
