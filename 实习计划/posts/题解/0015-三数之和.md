## 题目复述

给定整数数组 `nums`，找出所有和为 0 且不重复的三元组 `[nums[i], nums[j], nums[k]]`，下标互不相同。结果不能含重复三元组。注意本题只需返回数值，不需返回原始下标——这一点决定了能否排序。

## 初版思路 & 失败原因

- **做法一**：三重循环枚举所有 `(i, j, k)`，判断和是否为 0。
- **失败**：时间 O(n³)，超时。
- **做法二**：想沿用两数之和的哈希表思路，发现困难——三数关系时间复杂度再升一档，且去重难处理。

## 正解思路

核心：**排序 + 双指针**。

1. **排序**：题目只需返回数值、不需保留下标，可改原数组。排序后能用双指针按单调方向移动。
2. **固定 i，双指针扫 j/k**：`j = i+1`，`k = n-1`，计算 `s = nums[i] + nums[j] + nums[k]`。
3. **按和移动指针**：
   - `s > 0` → 太大，`k--`
   - `s < 0` → 太小，`j++`
   - `s == 0` → 记录结果，`j++`、`k--`
4. **去重**：i、j、k 各自跳过相邻重复值，否则会得到多个相同三元组。

## 代码

```python
def threeSum(self, nums):
    answer = []
    n = len(nums)
    nums.sort()
    if n < 3:
        return answer

    for i in range(n):
        if i > 0 and nums[i] == nums[i-1]:   # i 去重：跳过已用过的值
            continue
        j, k = i + 1, n - 1
        while j < k:
            s = nums[i] + nums[j] + nums[k]
            if s > 0:
                k -= 1
            elif s < 0:
                j += 1
            else:
                answer.append([nums[i], nums[j], nums[k]])
                j += 1
                k -= 1
                while j < k and nums[j] == nums[j-1]:   # j 去重
                    j += 1
                while j < k and nums[k] == nums[k+1]:   # k 去重
                    k -= 1
    return answer
```

## 复杂度分析

- 时间 O(n²)：外层 i 走 n 次，内层 j/k 双指针合计 O(n)。
- 空间 O(1)（不含输出；排序栈空间 O(log n)）。
- n = 数组长度。

## 关键点提炼

- 排序把"无序找三元组"降为"有序双指针"，是 O(n³) → O(n²) 的关键一跳。
- 去重必须在"已经用过该值之后"跳，不能在选之前跳，否则漏解。
- "返回数值而非下标" = 允许排序，这是选择双指针而非哈希表的前提信号。

## 踩坑反思

- **错误假设**：以为可以沿用两数之和的哈希表思路套到三数。
- **为什么错**：哈希表处理两数关系最优，三数关系时间复杂度再升一档，且结果去重困难。
- **纠正**：三数及以上的"有序可改原数组"问题，优先排序 + 双指针。
- **下次识别信号**：题目求"组合/三元组且不需保留下标" → 先想排序 + 双指针，别急着上哈希。

## 可迁移规律

- 题型归类：**有序数组上的 k 数之和**。
- 套路：排序 + 双指针，固定前 k-2 个、双指针扫最后两个。四数之和同法再加一层循环。
- 与两数之和的区别：两数之和要下标 → 哈希表；三数及以上要组合 → 排序双指针。
