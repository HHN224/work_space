1.爬楼梯：
假设你正在爬楼梯。需要 n 阶你才能到达楼顶。
每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

    斐波拉契数列，代码相当简单，一开始考虑了递归的做法，但是超时了，问题出在递归会大量重复调用，消耗大量性能，所以之后想了用数组，用哪个拿哪个就行了。

    我的做法，空间复杂度O(n)：
        dp = [0, 1, 2]          
        i = 3                   
        while i <= n:
            dp.append(dp[i-1] + dp[i-2])
            i += 1
        return dp[n]

    不浪费空间的做法,空间复杂度O(1)：
        if n <= 2:
            return n
        a, b = 1, 2  # 分别代表 f(i-2), f(i-1)
        for _ in range(3, n + 1):
            a, b = b, a + b
        return b


2.打家劫舍
你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。
给定一个代表每个房屋存放金额的非负整数数组，计算你 不触动警报装置的情况下 ，一夜之内能够偷窃到的最高金额。

    好吧，又是动态规划，我第一次又没想到，不如说我其实是想到了，但觉得解决不了所以否决了。我一开始想过用动态规划，因为这实际上和那个“最大子数组和”十分类似，但我一直没想到用什么方法来表示“空开任意格”。但这里确实犯了错误，就是对于这种根据前两个做选择的动态规划题，我并不关心两个取值之间空了多少，我只关心“前面的最优解”是多少，而这个“前面的最优解”其实就包含了“空开多个“这种选项，和当时我思考最大子数组和前面和后面要空开多少个的误区如出一辙，希望我下次再遇到类似的题可以独立做出来。

	    if not nums:
            return 0
        if len(nums) == 1:
            return nums[0]

        n = len(nums)
        dp = [0] * n
        dp[0] = nums[0]
        dp[1] = max(nums[0], nums[1])
        i = 2
        while i < n:
            dp[i] = max(dp[i-1], dp[i-2]+nums[i])
            i += 1
        return dp[n-1]
