1.什么是GIL，它对多线程的影响以及绕过方案：
    GIL是全局解释器锁，任意时刻，一个python进程里只能有一个线程能运行python字节码。
    以下两种情况会归还GIL：
        1，当一个线程执行超过100条字节码之后强制归还
        2，当一个线程进入io阻塞时立刻释放GIL，别的线程可以抢占GIL
    影响：
        1，CPU密集型的任务的时候对系统开销大，线程来回切换甚至可能不如直接单线程
        2，IO密集型的任务的时候性能有提升，多线程可以并发等待IO，效率高于单线程
    绕过方法：
        1，多进程multiprocessing：每个进程有独立的解释器和GIL，可以并行运行CPU计算；缺点：性能消耗大，进程之间通信消耗性能多
        2，协程asyncio：通过单线程的任务切换机制，提升在IO密集型任务的性能  
        3，用C/C++等语言写逻辑，封装成python拓展，CPU运行时手动释放GIL，如numpy，pandas，opencv

3.深浅拷贝：copy.copy() 与 copy.deepcopy() 的实现差异及使用场景:
    浅拷贝会创建一个全新的外层容器对象，容器内的元素都是旧对象的引用（外层是新开的内存，内部是旧内存）；
    深拷贝会递归地创建所有层级对象，外层容器，容器内的元素和所有嵌套对象都开辟新内存（外层和内层都是新开的内存）
    使用场景：
        浅拷贝：
            1，非嵌套对象，修改顶层也不会污染原数据，且开销很小
            2，嵌套对象，但不需要动内部嵌套对象
        深拷贝：
            1，嵌套对象，需要重新开一个副本防止污染原数据，但开销很大
            2，多层嵌套list/dict/自定义对象，后续可能改任意层级代码的情况

    装饰器本质是可调用对象，接受被装饰函数作为入参，包装时返回一个可调用对象，在调用这个可调用对象的时候才执行被装饰函数和其他被包装进去的逻辑。@warp(如果有参数)语法是func = warp(如果有参数)(func)的等价写法；
    类装饰器的实现要满足:
        1.__init__(self,func,如果有参数)逻辑要要有self.func = func
        2.要callable， __call__（self,*args, **kwargs）逻辑要实现返回包装后的内层函数

6.生成器机制：yield 关键字原理、生成器表达式、生成器 vs 迭代器 vs 列表推导式：
    yield关键字是在函数内部的一种特殊关键字，当函数拥有这个关键字的时候它会自动被识别成一个生成器；调用生成器函数不会执行代码，只会返回一个生成器对象。每次用next/for迭代时，函数从暂停位置运行到下一个yield，并且返回yield然后暂停现场，如果没有yield就抛出StopIteration异常；生成器属于迭代器，惰性逐个生成数据。迭代器属于可迭代对象，要实现__iter__()、__next__()。列表推导式用【】包裹，一次性把完整的列表存入内存，生成器惰性逐次生成不占用整块内存。

7.asyncio 事件循环：协程调度机制、async/await 语法糖、事件循环的执行流程：
    async是一个协程关键字，用来定义协程函数，调用协程函数则生成协程对象；await后面可以接 future/协程/Task。程序运行到await时，当前协程暂停并挂起，让出执行权；event loop调度其他协程执行；等到await对应的函数执行完毕，被挂起的协程丛await位置恢复运行，事件循环持续执行直到所有任务被执行完毕。

8.两数之和：
给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target  的那 两个 整数，并返回它们的数组下标。
你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。
你可以按任意顺序返回答案。

    一开始想的是两层循环遍历所有nums，然后全都相加看是不是等于target，等于就返回，是相当容易想到的思路，也确实很简单就做出来了，当然了，直接超时。后来实在想不到就去查了一下，发现可以用哈希表写，这里的思路是先用need = target - 现在循环到的数字，然后通过哈希表的低时间复杂度找对应的need，只有一个要注意的：hashmap[val] = idx哈希表必须是这种形式，不能反过来，因为我们要用数值need找对应的下标

        hashmap = {}
        for idx, val in enumerate(nums):
                need = target - val
                if need in hashmap:
                    return [idx, hashmap[need]]
                hashmap[val] = idx

9.三数之和：
给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。
注意：答案中不可以包含重复的三元组。

    当然了，一开始也是尝试了三层循环，理所当然地超时了。然后就想到上一题的哈希表，想用但是发现有点困难，因为这里需要三个数的关系，哈希表是两个数的关系最优，三个数的话时间复杂度又上了一个量级。并且这里其实不像上一题需要返回原始下标，只需要返回数字即可，所以是可以对原始数组进行各种操作的。所以最终发现要用“双指针”这种方法：一开始把原始数组按顺序排好，先对i循环，然后把第i个，第j个（j为i+1），第k个（k为末尾）数字求和。此时如果这个和大于0，就说明这里有人太大了，那么就只能降低k，所以把k往前一个数；如果这个和小于0，那么就是有人太小了，而在循环内又不能动i，就只能把j往后一个数；如果和等于0，当然就进入输出结果的路线了，这里记得j要往后迭代，k往前迭代。但是要注意把k的前一个和j的后一个和现在的值比较，如果相同就删掉，不然你就会得到多个相同的结果。最后如果jk重合就结束内层循环。每个外层循环都对i进行向右迭代。

        answer = []
        n = len(nums)
        nums.sort()

        if n < 3:
            return answer

        for i in range(n):
            if i>0 and nums[i] == nums[i-1]:
                continue
            
            j, k = i+1, n-1
            
            while j < k:
                s = nums[i] + nums[j] + nums[k]
                if s > 0:
                    k -=1
                elif s < 0:
                    j +=1
                else:
                    answer.append([nums[i],nums[j],nums[k]])
                    j +=1
                    k -=1
                    while j < k and nums[j] == nums[j-1]:
                        j +=1
                    while j < k and nums[k] == nums[k+1]:
                        k -=1

        return answer

            s = max(cur, s)
        return s
